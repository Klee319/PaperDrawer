# ホッパー連携

PaperDrawers はバニラのホッパーと連携してアイテムの自動搬入・搬出を実現します。

> **技術的な背景:** ドロワーはアイテムを PDC (PersistentDataContainer) に保存しており、バニラのインベントリシステムとは独立しています。そのため、ホッパーとの連携は PaperDrawers 独自の仕組みで実現しています。バニラのホッパー転送はブロックされ、代わりにプラグインがホッパーのインベントリを直接操作します。

## ホッパー → ドロワー（搬入）

ホッパーの排出口をドロワーに向けて設置すると、ホッパー内のアイテムが自動的にドロワーに搬入されます。

### 動作条件

- ホッパーの排出方向がドロワーブロックを向いていること
- ドロワーがそのアイテムを受け入れ可能であること（空のスロットがあるか、同じアイテムが格納されているスロットに空きがあること）
- ホッパーがレッドストーン信号で無効化されていないこと

### 搬入のルール

- 1サイクルにつき **1個** ずつ搬入
- ドロワーの各スロットを順番にチェックし、受け入れ可能な最初のスロットに搬入
- ボイドドロワーの場合はアイテムを消滅させる

```
設置例:

[ホッパー] → [ドロワー]
  (排出口をドロワーに向ける)
```

## ドロワー → ホッパー（搬出）

ドロワーの **真下** にホッパーを設置すると、ドロワーからアイテムが自動的に搬出されます。

### 動作条件

- ホッパーがドロワーの真下にあること
- ドロワーにアイテムが格納されていること
- ホッパーのインベントリに空きがあること
- ホッパーがレッドストーン信号で無効化されていないこと

### 搬出のルール

- 1サイクルにつき **1個** ずつ搬出
- アイテムが入っている最初のスロットから搬出
- ボイドドロワーからは搬出されない
- アイテムのメタデータ（エンチャント、カスタム名など）は搬出時に正しく復元される

```
設置例:

[ドロワー]
    ↓
[ホッパー]
```

## 仕分けドロワーの自動収集

[仕分けドロワー](Special-Drawers.md) は通常のホッパー連携に加え、隣接するホッパーからアイテムを **能動的に吸い取る** 機能を持っています。

### 通常ドロワーとの違い

| 機能 | 通常ドロワー | 仕分けドロワー |
|------|:-----------:|:------------:|
| ホッパーからの搬入（排出口がドロワー向き） | 対応 | 対応 |
| ドロワー下のホッパーへの搬出 | 対応 | 対応 |
| 隣接ホッパーからの自動吸い取り | 非対応 | **対応** |

### 自動吸い取りの詳細

- 隣接する **6方向**（上下左右前後）のホッパーを走査
- **設定済みスロット** にマッチするアイテムのみ吸い取り
  - アイテムが格納されているスロット
  - ドロワーキーでロックされたスロット
- 1サイクルにつき **1個** ずつ吸い取り
- レッドストーン動力を受けたホッパーは無視

### 実用的な設置パターン

#### パターン1: 横方向の仕分け

```
[チェスト] → [ホッパー] → [ホッパー] → [ホッパー] → ...
                 ↑             ↑             ↑
            [仕分けドロワー] [仕分けドロワー] [仕分けドロワー]
             (丸石ロック)    (鉄鉱石ロック)   (石炭ロック)
```

仕分けドロワーはホッパーに隣接していれば方向を問わず吸い取ります。

#### パターン2: 縦方向の仕分け

```
[ホッパー]
    ↓
[仕分けドロワー: 丸石]  ← ホッパーが隣接しているので吸い取り可能
    ↓ (ホッパー)
[仕分けドロワー: 鉄鉱石]
    ↓ (ホッパー)
[ボイドドロワー] ← 余りを処分
```

## レッドストーン制御

レッドストーン信号を受けたホッパーは無効化されます。これはバニラの動作と同じです。

- 搬入・搬出ともにレッドストーン信号で停止可能
- 仕分けドロワーの自動吸い取りも、動力を受けたホッパーからは行わない

これにより、レッドストーン回路を使ったアイテムフローの制御が可能です。

## パフォーマンスに関する設定

ホッパー連携のパフォーマンスは `config.yml` で調整できます。

```yaml
hopper:
  enabled: true          # ホッパー連携の有効/無効
  pull-interval-ticks: 4 # ホッパー処理の間隔（tick単位）
```

`pull-interval-ticks` のデフォルトは **4 tick** です（バニラのホッパー速度 8 tick より高速）。サーバー負荷が気になる場合は値を大きくしてください。
